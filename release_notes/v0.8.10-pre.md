
#### FEATURES

- Added a tool/function called `sub_agent` that is always available and can be used to query / prompt the LLM in a separate session with an isolated context. 
  - The prompt is sent to the same model in use by the current session
  - The agent runs without any history from the current session
  - The agent inherits all the current session tools and prompts
  - The agent returns the _session outline_, i.e. the first request and the last response, as a JSON array of messages.
  - Use with prompts like the following example:
    - `I want to document the modules in 'src/sucre' and create an 'ABOUT.md' file for each one. Use a sub-agent per module to do the analysis AND write the file.`
