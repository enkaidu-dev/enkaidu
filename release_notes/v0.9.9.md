#### NEW

- Add `/include audio_file PATH` command to include an audio file (WAV or MP3 only) in the next prompt.
- Add `AudioEditing` toolset with two tools: `write_audio_file` and `read_audio_file`

#### IMPROVEMENTS

- Built-in tools that return images are now processed to ensure that the model receives the images after the tool call response as a separate response with the correct message structure.
- Modify `ImageEditing` toolset, renaming `create_image_file` to `write_image_file`, adding optional `overwrite` flag.

#### FIXES

- Fix `read_image_file` so that now a user can prompt the model to read a file and analyze it.

#### UNDER THE HOOD

- Added a way for tools to return a `LLM::Function::Reply` that gathers attachments so that the protocol can appropriately handle the response when images and files are included
